*Sales Analytics Dashboard – Sun Pharma Case Study

## Project Objective
The objective of this project is to clean the provided sales dataset,
analyze it, and build an interactive dashboard to help management take
data-driven business decisions.

*Dataset Description
The dataset contains sales inquiry information such as:
- Inquiry Number
- Timestamp
- category
- Update Remarks
- Company Name
- Sales Person Name
- Sales Stage
- Estimated Order Value
- Budget, Quantity
- Lead Source and Follow-up details and etc...

The data was provided as part of an analytics assignment.

*Tools & Technologies Used
- Python (Pandas) – Data Cleaning
- Google Colab – Coding Environment
- Power BI – Dashboard Creation
- Microsoft Excel – Raw & Cleaned Data
- GitHub – Project Submission

*Data Cleaning Steps
- Removed duplicate inquiries
- Standardized column names
- Converted date columns to proper date format
- Handled missing values in budget and order value
- Ensured correct data types for numerical fields

The complete data cleaning code is available in `data_cleaning.py`.

*Key KPIs in Dashboard
1. Estimated Business Generated from Sun Pharma
2. Most Potential Client (by Estimated Revenue)
3. Best Salesperson (most clients & highest expected business)
4. Salesperson Handling the Biggest Client
5. Lead Conversion Rate (Won vs Total Leads)

*Key Business Insights
- A small number of clients contribute a large portion of total revenue.
- Certain salespersons consistently handle high-value clients.
- Lead conversion rate highlights scope for improving follow-ups.
- Sun Pharma contributes significantly to overall expected business.

*Repository Contents
- `data_cleaning.py` – Python script for data cleaning

*Conclusion
This dashboard provides a clear view of sales performance, client potential,
and salesperson effectiveness, enabling management to make informed decisions.
